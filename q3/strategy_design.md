# 题目 3：多因子中性策略设计

## 1. 核心投资逻辑

加密货币市场（尤其是永续合约市场）在小时级别呈现出**动量与均值回归共存**的微观结构特征：

- **短期动量**（1-6h）：资金流入惯性、FOMO 情绪驱动，方向性延续概率显著高于随机游走
- **中期均值回归**（12-24h）：资金费率套利、期现价差收敛使得过度偏离后存在回调压力
- **量价共振**：成交量放大伴随价格突破时趋势延续性更高；缩量反弹则大概率回落

因此，本策略在截面上**做多相对强势、低波动率币种，做空相对弱势、高波动率币种**，通过多因子合成信号兼顾趋势捕捉与风险控制，同时保持**市场中性**（总敞口为零），规避系统性 Beta 风险。

---

## 2. 因子来源与预处理

### 2.1 因子体系

| 类别 | 因子 | 计算方式 | 预期方向 |
|------|------|---------|---------|
| 动量 | log_ret_1h/3h/6h | 对数收益率 | 正（短期延续） |
| 反转 | log_ret_12h/24h | 对数收益率 | 负（中期均值回归） |
| 波动率 | vol_24h, rv_24h | 滚动标准差 / RMS | 负（高波动→高风险） |
| 量价 | volume_over_ma_24h, volume_chg_1h | 量相对均值、环比变化 | 正（放量趋势更强） |
| 技术 | RSI_14, MACD_hist, BB_width | 标准技术指标 | 见下注 |
| 趋势质量 | TC_n6/n12/n24 | 价格-时间线性拟合 R² | 正（趋势越线性方向越明确） |
| 价格均值 | PWMA_10 | 帕斯卡权重 MA 偏离度 | 做多偏离向上的币种 |
| 振荡 | CFO_14 | Chande 预测振荡器 | 正（正值=价格高于线性预测，延续性） |

> RSI 方向：RSI<30 反转做多，RSI>70 短期可做空（但在趋势行情中反转信号失效，需结合 MACD 过滤）
>
> MACD_hist 正值表示短期上升动量，与 log_ret 方向一致，取正向权重

### 2.2 预处理流程（每小时截面）

```
1. Winsorize (双侧 1%): clip 极端值，防止单一异常数据主导
2. 截面 Z-Score: (factor - cross_mean) / cross_std，使各因子可比
3. 填充 NaN: 用截面中位数填充（保留样本数量）
4. 因子中性化（可选）:
   - 针对有明显 Beta 暴露的因子（如 log_ret_1h），对市场基准收益率做残差正交化
   - 从而剥离"仅仅是跟涨跌"的成分，保留真正的截面区分度
```

## 3. 因子合成与模型

### 3.1 线性合成（基准，优先）

```python
composite_score = sum(w_k * factor_k_normalized)
```

权重 `w_k` 通过以下方式确定（优先级递减）：

1. **IC 加权**：`w_k = mean(IC_k) / sum(|mean(IC_k)|)`（ICIR 加权变体更稳定）
2. **Ridge 回归**：`min ||y - Xw||² + alpha * ||w||²`（L2 正则避免过拟合）
3. **等权**（基线）：验证因子本身有效时使用

### 3.2 非线性增强（可选，需更严格验证）

```python
model = LGBMRegressor(n_estimators=200, max_depth=4, learning_rate=0.05)
```

- **注意**：LightGBM 在样本量不足时容易过拟合；**要求 OOS 验证周期 ≥ 6 个月**
- 建议：线性模型作为主力，LGB 作为第二模型，ensemble 混合（`0.7*Ridge + 0.3*LGB`）

## 4. 持仓构建（截面多空）

```
每小时 t：
1. 计算截面综合得分 S_{i,t} for all symbols i
2. 按得分排序，取前 20% long，后 20% short
3. 权重：等权（简单）或按得分大小加权（更稳健）
4. 多头权重之和 = 空头权重之和 = 0.5（名义）
   => 净敞口 = 0，实现市场中性
5. 每小时调仓（Rebalance at close of each bar）
```

---

## 5. 回测方式（Walk-Forward）

```
时间轴: 2023-01 → 2025-12（24 个月）
训练窗口: 滚动 3 个月
验证窗口: 紧随其后 1 个月（OOS）
每月月末重新拟合权重，下月运行

Walk-Forward 折叠示意：
  Train [Jan-Mar] -> Test [Apr]
  Train [Feb-Apr] -> Test [May]
  ...
```

### 关键评估指标

| 指标 | 目标 |
|------|------|
| IC（截面） | > 0.02（按月平均） |
| ICIR | > 0.3 |
| 年化 Sharpe | > 1.5（多空组合） |
| 最大回撤 | < 20% |
| 月度胜率 | > 55% |
| 年化换手率 | < 2000%（控制交易成本） |

## 6. 风险管理

### 6.1 持仓限制
- **单仓位上限**：任何单一币种权重 ≤ 10%（防止集中风险）
- **行业/Beta 中性**：对主流 Altcoin 与 BTC 的相关性做因子中性化，确保多空两端 Beta 抵消

### 6.2 回撤熔断
- **日内回撤 > 5%**：减仓 50%，暂停开新仓 4 小时
- **累计回撤 > 15%**：完全平仓，停策略，人工审查
- **单因子失效检测**：当某因子 30 天滚动 IC < -0.01（持续负向），自动将该因子权重降至 0

### 6.3 流动性约束
- 仅交易 24h 成交额 > $50M 的币种（防止滑点过大）
- 单次下单量不超过该时段 1 分钟均量的 5%
- 将预估滑点（约 0.05-0.1 bps 每次交易）纳入回测成本

---

## 7. 实盘执行

### 7.1 订单执行
- **优先使用限价单**（Maker 端），避免 Taker 手续费
- 若 1 分钟内未成交，降价 0.01% 重挂（最多 3 次），之后市价强制成交
- 建议在整点后 1-2 分钟执行（等待开盘价稳定）

### 7.2 资金费率风险
- Binance 永续合约每 8 小时结算资金费率
- **做多高费率**（正费率）币种会产生额外成本：每日统计预期资金费率成本
- 策略每小时调仓时应将`资金费率 * 持仓时间`纳入持仓成本估算
- 若预期费率成本 > 预期信号收益的 30%，减半该方向仓位

### 7.3 操作流程（每小时）
```
T-2min : 计算新一轮因子得分（基于最新 K 线）
T-1min : 生成目标仓位
T+0min : K 线收盘，开始执行订单
T+2min : 检查成交情况，未成交订单降价补单
T+5min : 记录实际成交均价，计算滑点
```

---

## 8. 关键风险提示

1. **过拟合风险**：加密因子在牛熊切换时失效率高，**强烈建议 OOS 验证 ≥ 6 个月**
2. **流动性风险**：小币种在极端行情下买卖价差扩大 10x，务必使用流动性过滤
3. **相关性集中风险**：Altcoin 与 BTC 相关性在恐慌时会趋向 1，中性假设失效，需设置相关性监控熔断
4. **信号衰减**：过去有效因子可能因市场参与者学习而失效，需定期（每季度）重新检验因子显著性

---

## 9. 总结

本策略的核心优势在于：
- **截面多空**：通过做多强势、做空弱势币种天然对冲市场方向风险
- **多因子分散**：12 个以上因子线性合成，任何单因子失效不会导致策略崩溃
- **Walk-Forward 回测**：杜绝未来数据泄漏，评估结果更接近实盘表现
- **严格风控**：多层熔断机制，将单次策略失效损失控制在可接受范围

潜在挑战：加密市场 regime 切换频繁，因子有效性不稳定，实盘中需要持续的因子有效性监控和权重动态调整机制。
